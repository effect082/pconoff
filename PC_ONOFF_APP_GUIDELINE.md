# PC 출퇴근(ON/OFF) 모니터링 시스템 완벽 구현 가이드라인

본 문서는 AI가 읽고 즉시 "PC ON/OFF 기능 구현 앱"을 완벽하게 재구성(구현)할 수 있도록 설계된 최종 마스터 가이드라인입니다. 모든 프론트엔드, 데스크탑 앱, 백엔드 구성 요소는 이 문서의 규칙을 엄격하게 따라야 합니다.

## 1. 프로젝트 개요 및 아키텍처
- **목적**: Windows PC의 부팅(출근)과 종료(퇴근) 시간을 정확히 기록하고, 관리자 대시보드를 통해 직원의 근태를 통합 관리하는 시스템.
- **기술 스택**:
  - **백엔드 (DB & API)**: Google Apps Script (GAS) + Google Spreadsheet (무료 호스팅 및 DB 대체)
  - **데스크탑 클라이언트 (감시 에이전트)**: Electron (Node.js) + PowerShell
  - **프론트엔드 (관리자 Web)**: HTML / CSS / Vanilla JS

## 2. 백엔드 구현 가이드 (GAS & Spreadsheet)
- **단일 파일 구조**: `Code.gs`에서 `doGet(e)`, `doPost(e)` 파라미터(`action`) 분기를 통해 REST API 형태로 클라이언트 요청 처리.
- **DB 자동화 (`initSheets`)**: 최초 연결 시 `Users`, `Logs`, `AdminSettings` 시트를 자동 생성 (초보 관리자 설정 간소화).
- **핵심 API 엔드포인트 로직**:
  - `recordBoot`: 해당 날짜의 부팅 시간(출근) 기록.
  - `recordOff`: 종료 시간(퇴근) 기록. *주의: 구글 시트가 임의로 Date로 파싱하지 않도록 셀 포맷을 문자열(`@`)로 강제 지정.*
  - 기타 API: 사용자 등록/로그인, 관리자 로그인, 대시보드 데이터(근태 목록, 회원 목록) 송수신.

## 3. 데스크탑 앱(클라이언트) 핵심 구현: 다중 방어 종료 감시 로직
가장 중요한 "퇴근 시간 누락 방지"를 위해 다음 5단계 다중 방어 로직을 모두 구현해야 합니다.
1. **내장 이벤트 감지**: `powerMonitor.on('shutdown')`, `app.on('session-end')`, `SIGTERM`, `SIGINT` 등을 통한 OS 시그널 감지.
2. **이벤트 로그 감시**: PowerShell `EventLogWatcher`를 백그라운드로 실행해 Windows 시스템 종료 로그(Event ID 1074, 42, 6006 등) 실시간 감지.
3. **Detached 비동기 통신**: 앱 종료 과정에서 메인 통신이 끊기는 것을 방지하기 위해 `child_process.spawn` (Detached 모드)으로 `curl.exe`와 `Invoke-RestMethod` 동시 발송. (Fallback 로직)
4. **Heartbeat (가장 중요)**: 예기치 않은 강제 종료/정전을 대비해 **30초 간격으로 현재 시간을 `offTime`으로 서버에 전송하는 타이머(`setInterval`) 가동**. (서버는 최신 시간으로만 덮어쓰기)
5. **로컬 캐시 보정**: `shutdown_cache.json`에 현재 시간을 저장하고, 부팅 시 `wevtutil qe System /f:xml` 분석을 통해 정상 종료 시간을 역추적하여 누락된 퇴근 시간 복구.

## 4. 관리자 대시보드 (Admin Web) UI/UX 상세 명세 (핵심 기능)

### 4.1. 전역 UI 디자인 방향 (필수 준수)
모든 프론트엔드 요소(Admin Web 및 Desktop App UI)는 다음의 디자인 가이드라인을 엄격히 따릅니다.
- **스타일 (Style)**: 글래스모피즘(Glassmorphism) 스타일 적용 (반투명 배경, 배경 블러 효과, 은은한 테두리를 활용한 입체적 디자인).
- **색상 (Color)**: 흰색(White)과 파란색(Blue) 계열 중심. 채도가 너무 강하거나 과도한 그라데이션은 지양하여 눈이 편안하고 깔끔한 느낌 제공.
- **톤앤매너 (Tone & Manner)**: 전체적으로 모던하고 세련된(Modern & Sleek) UI 지향.
- **타이포그래피 (Typography)**: 기본 폰트로 **Paperlogy (페이퍼로지)** 서체 적용.

### 4.2. 대시보드 레이아웃 개요
관리자가 직원의 근태와 계정을 체계적으로 관리할 수 있도록 2개의 메인 탭(근태 현황 / 회원 관리)으로 구성합니다.

### 4.3. 공통 상단 영역 (Header)
- 탭 네비게이션: [근태 현황], [회원 관리] 탭 전환 기능.
- 우측 상단 유틸리티: [비밀번호 변경], [로그아웃] 버튼.

### 4.4. 근태 현황 탭 (Attendance Status)
- **보고서 타이틀**: `[YYYY-MM-DD] 전체 PC 접속시간 현황 보고서` 형태로 선택된 날짜 동적 표기.
- **검색 및 필터링 (Search & Filter)**:
  - `날짜 선택` (Date Picker): 조회할 특정 날짜 지정.
  - `직원 검색` (텍스트 입력): 특정 직원의 이름으로 리스트 필터링.
  - `조회` 버튼: 입력된 필터 조건으로 테이블 데이터 갱신.
- **보고서 인쇄 및 직인 기능 (Print & Seal)**:
  - `보고서 인쇄` 버튼: 현재 조회된 테이블을 출력(브라우저 인쇄 팝업)하는 기능.
  - `직인 업로드`: 인쇄 시 우측에 표시될 기관/관리자의 직인 이미지(png/jpg)를 파일 선택으로 업로드하고, 화면 우측에 미리보기를 렌더링.
- **근태 데이터 테이블**:
  - 컬럼: `날짜`, `이름`, `출근(PC 켠 시간)`, `퇴근(PC 끈 시간)`.

### 4.5. 회원 관리 탭 (Member Management)
- **목적**: 등록된 직원의 정보, 부서/직급, 재직 상태 및 보안을 관리.
- **회원 관리 데이터 테이블**:
  - 컬럼: `부서`, `이름`, `직급`, `상태`, `비밀번호 재설정`.
  - `상태 변경`: 드롭다운(Select) 요소로 구현하여 '재직', '퇴사' 등의 상태값을 즉각적으로 변경 및 DB에 저장.
  - `비밀번호 초기화`: [비밀번호 초기화] 버튼 클릭 시, 해당 직원의 비밀번호를 초기화(기본 비밀번호 부여 등) 처리하는 기능.

## 5. 부가 기능: 퇴근 독려 및 시간외근무 관리 (Admin & Client 연동)
관리자가 시간외근무 미신청자에게 퇴근을 유도하는 기능을 직접 설정하고 운영합니다.
- **알림 시간 및 안내문 설정 (Admin Web)**: 관리자 대시보드 내에 퇴근 독려 알림이 작동할 기준 '시간/분'과 노출될 '안내문 텍스트'를 작성할 수 있는 UI 제공. 이 설정값은 `AdminSettings` 시트에 저장됨.
- **조건부 알림 팝업 발송 (Desktop 앱)**: 설정된 알림 시간에 도달 시, 데스크탑 앱은 서버와 통신하여 해당 직원의 '시간외근무 신청 여부'를 확인. 미신청자로 확인될 경우, 관리자가 작성한 커스텀 안내문을 PC 바탕화면 우측 하단에 알림(팝업)으로 노출하여 퇴근을 독려.

## 6. 초기 설치 및 배포(시작프로그램) 가이드라인
- **최초 실행 화면 (Setup Wizard)**: 데스크탑 앱 및 관리자 웹 최초 진입 시, **"GAS 웹앱 URL"**을 입력하는 명확한 입력란 제공. [URL 검증 테스트] 통과 시에만 로컬 스토리지 또는 JSON 파일에 저장.
- **시작프로그램 자동 등록**: Electron의 `app.setLoginItemSettings()`를 사용해 클라이언트 설치 시 윈도우 시작프로그램에 자동으로 등록.
- **UX 최소화**: 관리자가 직원 PC 세팅 시 "URL 입력 -> 직원 이름/비밀번호 로그인" 두 단계만 거치면 모든 설정과 백그라운드 모니터링이 즉시 시작되도록 구현. 트레이(Tray) 아이콘으로 숨김 동작 지원.
